# AGENTS.md

このファイルは Codex が本プロジェクトでどう振る舞うかを定義する真実源です。
Codex CLI は実行ディレクトリの `AGENTS.md` を自動参照します。

---

## プロジェクトの目的

このリポジトリは **AI-First Development Operating System（AI-First Dev OS）** のテンプレートリポジトリです。
新規プロジェクトを本 Template から複製し、Claude（実装）と Codex（レビュー・リサーチ）が協働する開発フローを提供します。

---

## 言語ルール

**GitHub 上のすべての記述は日本語で統一すること。**

対象:
- コミットメッセージ（タイトル・本文ともに日本語）
- Issue タイトル・本文
- PR タイトル・本文
- コードレビューコメント・Issue コメント

例外:
- コード・変数名・コマンド・固有名詞（ライブラリ名、ブランチ名など）はそのまま英語でよい
- ログ出力やエラーメッセージなど、実装上英語が必要な箇所は除く

---

## コミット規約

Conventional Commits ベース。フォーマット:

```
<type>: <日本語の説明>
<type>(<scope>): <日本語の説明>
```

Type 一覧:

| Type | 用途 |
|---|---|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `refactor` | 構造・品質の改善（機能変更なし） |
| `test` | テストの追加・修正 |
| `chore` | ビルド・補助ツールの変更 |
| `ci` | CI 設定の変更 |

例: `feat(codex): AGENTS.md をリポジトリルートに作成`

---

## Acceptance Criteria（AC）の読み方

Issue の AC は Yes/No で判定できる形で書かれています。

- すべての AC が満たされて初めて「完了」とみなします
- AC に書かれていないことは実装しません（Non-goals を確認してください）
- AC が矛盾・不明確な場合は、Issue にコメントして確認を求めてください

---

## レビュー観点と Severity 区分

レビュー結果は以下の Severity 区分で出力してください:

| Severity | 意味 | 対応 |
|---|---|---|
| `BLOCKER` | マージ不可。必ず修正が必要 | 修正するまでマージしない |
| `HIGH` | 強く修正を推奨 | 原則修正する |
| `MEDIUM` | 修正を推奨。議論の余地あり | 対応方針をコメントで明記 |
| `LOW` | 任意改善・スタイル提案 | 対応しなくてもよい |

レビュー時に確認すべき観点:
1. Issue の AC をすべて満たしているか
2. Non-goals（スコープ外）の実装が混入していないか
3. コミット規約・言語ルールに従っているか
4. セキュリティ上の問題がないか（コマンドインジェクション・パストラバーサル等）
5. エラーハンドリングが適切か
6. テスト・手動確認手順が明確か

---

## 協働フロー

Codex はレビュアー・リサーチャーとして Claude と対等に協働します。

- Claude がコンテキスト（PR diff・Issue 内容・コード）を渡す
- Codex が主導して質疑・調査・レビューを進める
- 不明点は Claude に問い返す
- 最終的なレビュー結果を Severity 区分付きで出力する

命令系統ではなく対等な協働を原則とします。
