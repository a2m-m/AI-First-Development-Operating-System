# 要件定義書

**プロジェクト名**: Diffairy  
**作成日**: 2026年3月10日  
**バージョン**: 1.5.0

---

## 1. プロジェクト概要

### 背景・課題

AIコーディングツール（Claude Code など）を使った開発では、AIが自動的に複数ファイルを書き換えるため、開発者がどのような変更が行われたか把握しにくい状況が生まれている。コミット単位での差分確認は変更が大きすぎてキャッチアップが困難であり、「とりあえずOK」と変更を承認してしまうケースが多い。

### 解決したいこと

- ファイルが保存されるたびに、その差分（diff）をAIがわかりやすく日本語で解説して通知する
- アプリを閉じていても監視・通知が継続する
- 通知内容やコードについてそのままチャットで質問・深掘りできる
- ターミナルを一切使わず、Mac の普通のアプリとして起動・操作できる

### ゴール

> **「Dock のアイコンをクリックして、あとは開発するだけ。何が変わったかを即座に知れて、わからないことはそのままチャットで聞ける状態にする」**

## 3. アプリ形態

| 項目 | 内容 |
|------|------|
| 形式 | macOS ネイティブアプリ（`.app` バンドル） |
| 開発言語 | **Swift + SwiftUI** |
| 起動方法 | Dock に登録してクリック、または Finder からダブルクリック |
| 常駐方法 | Dock にアイコンが表示される通常のウィンドウアプリ |
| バックグラウンド監視 | ウィンドウを閉じても・アプリを終了しても監視継続 |

---

## 4. 通知の仕組み

通知は **3つのチャンネル** を持ち、それぞれ設定画面で個別に ON/OFF できる。

```
diff 発生
   │
   ├──→ ① アプリ内通知（常に記録）
   ├──→ ② macOS ネイティブ通知（設定でON/OFF）
   └──→ ③ LINE 通知（設定でON/OFF・要APIキー設定）
```

---

## 5. バックグラウンド監視

| ケース | 動作 | 実装方針 |
|--------|------|---------|
| ウィンドウを閉じた | 監視継続・Dock アイコンは残る | `applicationShouldTerminateAfterLastWindowClosed` を `false` に設定 |
| アプリを終了（Cmd+Q） | 監視停止の確認ダイアログを表示 | `applicationShouldTerminate` で制御 |
| ログイン時自動起動 ON | Mac 起動時にバックグラウンドで自動起動 | `SMAppService.mainApp` |
| バックグラウンド中の通知 | macOS ネイティブ通知で届く | `UNUserNotificationCenter` |

## 6. 複数フォルダ監視

| 項目 | 内容 |
|------|------|
| 登録数 | 上限なし（実用的には 10 フォルダ程度を想定） |
| 管理方法 | 設定タブにフォルダリストを表示し、追加・削除・有効/無効の切り替えができる |
| フォルダごとの設定 | 監視 ON/OFF をフォルダ単位で切り替えられる |
| 履歴の区別 | 通知履歴タブでリポジトリごとにフィルタリングできる |

│                                                          │

### 画面①：設定タブ

| セクション | UI要素 | 内容 |
|-----------|--------|------|
| 監視フォルダ | フォルダリスト | 登録済みフォルダを一覧表示。ON/OFF トグル付き |
| | 追加ボタン | `NSOpenPanel` でフォルダを選択して追加 |
| | 削除ボタン | 選択したフォルダをリストから削除 |
| 通知設定 | macOS通知 | `Toggle` で ON/OFF |
| | LINE連携 | `Toggle` で ON/OFF（OFF 時は入力欄がグレーアウト） |
| | LINE チャンネルアクセストークン | `SecureField` |
| | LINE ユーザーID | `TextField` |
| AI設定 | Gemini APIキー | `SecureField` |
| | 使用モデル | `Picker` でモデルを選択（例：`gemini-2.0-flash`、`gemini-2.0-pro` 等）|
| | モデル一覧の更新ボタン | Gemini API から利用可能なモデル一覧を取得して選択肢を更新する |
| 監視設定 | 除外パターン | リスト ＋ 追加・削除ボタン |
| | デバウンス秒数 | `Stepper` |
| システム | ログイン時自動起動 | `Toggle` |
| | 保存ボタン | 設定を保存 |

### 画面②：通知履歴タブ

| UI要素 | 内容 |
|--------|------|
| 通知ログ一覧 | 日時・リポジトリ名・ファイル名・解説冒頭のリスト（新しい順） |
| 詳細パネル | 選択ログの diff全文・解説全文を右ペインに表示 |
| 「このdiffについて質問する」ボタン | 選択したログをコンテキストとしてチャット画面に引き継ぐ |
| リポジトリ絞り込み | ドロップダウンでリポジトリ別に絞り込み |
| 日付フィルター | 「今日」「今週」「すべて」のセグメント切り替え |
| キーワード検索 | ファイル名・解説文を全文検索 |
| ログ削除 | スワイプまたは削除ボタンで個別削除 |

### 画面③：チャットタブ ⭐️ 新規追加

AI とのチャット画面。通知内容・コードへの質問が自由にできる。

| UI要素 | 内容 |
|--------|------|
| チャット履歴エリア | メッセージを吹き出し形式で表示（上から時系列順） |
| コンテキスト表示 | 「このdiffについて質問する」から遷移した場合、画面上部に参照中のファイル名・diff・解説を折りたたんで表示 |
| テキスト入力欄 | 複数行対応の入力フィールド（Shift+Enter で改行、Enter で送信） |
| 送信ボタン | メッセージを送信して Gemini に回答を依頼する |
| ローディング表示 | Gemini が応答中はタイピングアニメーション（ドット点滅など）を表示 |
| 新しいチャットボタン | 会話履歴をリセットして新規チャットを開始する |
| チャット履歴の保存 | 過去のチャットセッションを一覧から呼び出せる（日時・最初のメッセージで識別） |

### 画面④：プロンプト編集タブ

| UI要素 | 内容 |
|--------|------|
| プロンプトエディター | `TextEditor` で全文編集 |
| 変数説明 | `{filepath}`・`{diff}`・`{reponame}` の説明を表示 |
| リセットボタン | デフォルトプロンプトに戻す（確認ダイアログあり） |
| 保存ボタン | 編集内容を保存 |

## 8. チャット機能の詳細仕様

### 8-1. 基本動作

チャットタブでユーザーが自由にテキストを入力し、Gemini API が応答を返す。会話は複数ターン継続できる（直前の会話履歴をAPIに渡して文脈を保持）。

### 8-2. コンテキストの引き継ぎ

通知履歴タブの「このdiffについて質問する」ボタンを押すと、以下の情報がチャットのシステムプロンプトに自動でセットされた状態でチャットタブが開く。

```
【参照中のdiff情報】
- リポジトリ名: {reponame}
- ファイル名: {filepath}
- 変更日時: {datetime}
- diff内容: {diff}
- AIの解説: {explanation}

このdiff情報を前提として、ユーザーの質問に答えてください。
```

ユーザーは「このファイルって何をしてるの？」「この変更は何のため？」のように自然に質問できる。

### 8-3. 質問できる内容の例

| 質問の種類 | 例 |
|-----------|-----|
| diff への質問 | 「この変更って何のためにやったの？」「どこが一番重要な変更？」 |
| コードへの質問 | 「このファイルの役割を教えて」「この関数は何をしてる？」 |
| 自由な質問 | 「Reactのコンポーネントって何？」「エラーが出たんだけどどうすれば？」 |

### 8-4. チャット履歴の管理

| 項目 | 内容 |
|------|------|
| 保存単位 | 1セッション（「新しいチャット」で区切られる） |
| 保存内容 | 日時・最初のメッセージ・全会話内容 |
| 保存先 | SwiftData（通知履歴と同じDB） |
| 呼び出し | 過去のセッションをサイドバーまたはリストから選択して再表示 |

## 10. 非機能要件

| # | 要件 | 内容 |
|---|------|------|
| N-01 | レスポンスタイム | ファイル保存から通知まで **10秒以内** |
| N-02 | 安定性 | 長時間・バックグラウンド動作でも監視が止まらないこと |
| N-03 | セキュリティ | APIキーは macOS **Keychain** に保存 |
| N-04 | 保守性 | 設定・プロンプトはすべてUI上から変更可能 |
| N-05 | 動作環境 | macOS Ventura（13）以降、Apple Silicon / Intel 両対応 |
| N-06 | データ永続性 | 通知履歴・チャット履歴は SwiftData で保存 |
| N-07 | 配布性 | `.app` バンドルのみで動作、追加ランタイム不要 |

| 分類 | 技術 | 用途 |

| 言語 | Swift 5.9+ | メインロジック・UI |

## 14. セットアップ手順（ユーザー向け）

1. `Diffairy.app` を `/Applications` フォルダに移動
2. Dock に追加してクリックで起動
3. 設定タブで以下を設定して「保存」：
   - 監視したいリポジトリフォルダ（複数追加可）
   - Gemini APIキー
   - macOS通知 / LINE連携 の ON/OFF
4. 「監視開始」ボタンを押すだけ 🎉
