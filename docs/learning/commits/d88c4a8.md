# Learning Content: PR Updates

## Summary
この変更は、Claude Code を利用したコードレビューの統合方法を Model Context Protocol (MCP) ベースの方式から、直接ターミナルコマンド (`claude -p`) を使用し、その出力をファイルにリダイレクトする方式へと変更するものです。主な目的は、Antigravity (Gemini) エージェントの会話コンテキストがレビュー結果によって圧迫されるのを防ぐことにあります。これにより、MCPに関連する設定ファイル (`.mcp.json`) や移行ガイドが削除され、よりシンプルでコンテキスト管理が容易な統合フローに移行しました。

## Deep Dive
- **MCP連携の完全廃止**: これまでClaude CodeをMCPサーバーとして登録し、Antigravityからツールとして呼び出す方式が完全に廃止されました。具体的には、MCPサーバーの定義を含む`.mcp.json`ファイルと、その移行を解説していた`docs/claude-code-mcp-migration-guide.md`が削除されています。これは、以前のMCP移行が何らかの運用上の課題を抱えていたか、または、より直接的な制御が必要と判断されたことを示唆しています。
- **`claude -p`コマンドへの回帰と出力リダイレクト**: 新しいレビュー依頼方法は、Antigravityが直接ターミナルで`claude -p`コマンドを実行し、レビュープロンプトを渡す形に戻りました。特に重要なのは、その出力を一時ファイル（例: `/tmp/claude_review_result.json`）にリダイレクトする点です。これにより、レビュー結果が直接Antigravityの会話コンテキストに流れ込むことを防ぎ、トークン消費の最適化とコンテキストの管理を改善します。
- **Claude Codeへの厳格な出力形式要求**: `claude/CLAUDE.md`および`agents/rules/rules.md`において、Claude Codeが`claude -p`経由でレビュー依頼を受けた場合、「プロンプトで指定された出力形式（JSON等）に厳密に従い、他の不要なテキストやMarkdownコードブロックの枠（```json 等）を含めないこと」という指示が明確化されました。これは、ファイルリダイレクトされたJSONをAntigravityが正確にパースするために不可欠な要件です。
- **コンテキストクリアコマンドの導入**: `claude -p "/clear"`というコマンドがレビュープロンプトの前に実行されるようになりました。これは、Claude Codeが前回の対話履歴を保持しない独立した状態でレビューを行うことを保証し、毎回クリーンな状態からレビューを開始することで、レビューの再現性と信頼性を高めます。
